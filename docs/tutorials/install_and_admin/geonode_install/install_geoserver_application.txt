=================
Install GeoServer
=================

In this section we are going to setup GeoServer for GeoNode. GeoServer will run inside
`Tomcat` servlet container.

Download and Setup
==================

#. First download GeoServer by help of Paver. (Make sure your virtuualenv is activated!)

    .. code-block:: bash

        source /home/geonode/.venv/my_geonode/bin/activate
        cd /home/geonode/my_geonode/ 
        paver setup

#. You've already installed `Tomcat 8` in the first section of the training. Before you deploy GeoServer stop the running Tomcat instance.

    .. code-block:: bash

        sudo service tomcat8 stop

#. Now copy the downloaded GeoServer archive inside Tomcat's webapps folder

    .. code-block:: bash

        sudo cp /home/geonode/my_geonode/downloaded/geoserver-2.*.war /var/lib/tomcat8/webapps/geoserver.war

#. And update catalina.properties

    .. code-block:: bash

        sudo sed -i -e 's/xom-\*\.jar/xom-\*\.jar,bcprov\*\.jar/g' /var/lib/tomcat8/conf/catalina.properties



#. To keep things tidied move ``GEOSERVER_DATA_DIR`` to an external location

    .. code-block:: bash

        sudo mkdir -p /data/geoserver-data;
        sudo mkdir -p /data/geoserver-logs;
        sudo mkdir -p /data/gwc_cache_dir;
        sudo cp -Rf /home/geonode/my_geonode/geoserver/data/* /data/geoserver-data/;
        sudo chown -Rf tomcat8:tomcat8 /data/geoserver-data/;
        sudo chown -Rf tomcat8:tomcat8 /data/geoserver-logs/;
        sudo chown -Rf tomcat8:tomcat8 /data/gwc_cache_dir/;

#. Set default `Java` settings

    You need to edit the ``/etc/default/tomcat8`` file
    
    .. code-block:: bash

        sudo vim /etc/default/tomcat8
        
    Make sure ``JAVA_OPTS`` are configured as follows

    .. code-block:: yaml
    
        GEOSERVER_DATA_DIR="/data/geoserver-data"
        GEOSERVER_LOG_LOCATION="/data/geoserver-logs/geoserver.log"
        GEOWEBCACHE_CACHE_DIR="/data/gwc_cache_dir"
        GEOFENCE_DIR="$GEOSERVER_DATA_DIR/geofence"

        JAVA_OPTS="-Djava.awt.headless=true -Xms512m -Xmx2048m -Duser.timezone=GMT -Dorg.geotools.shapefile.datetime=true -XX:+UseConcMarkSweepGC -XX:ParallelGCThreads=4 -Dfile.encoding=UTF8 -Duser.timezone=GMT -Djavax.servlet.request.encoding=UTF-8 -Djavax.servlet.response.encoding=UTF-8 -DGEOSERVER_DATA_DIR=$GEOSERVER_DATA_DIR -Dgeofence.dir=$GEOFENCE_DIR -DGEOSERVER_LOG_LOCATION=$GEOSERVER_LOG_LOCATION -DGEOWEBCACHE_CACHE_DIR=$GEOWEBCACHE_CACHE_DIR"

    .. warning:: Double check memory options ``-Xms512m -Xmx2048m`` are compatible with your VM available RAM


#. Restart `Tomcat 8` service

    .. code-block:: bash

        sudo service tomcat8 restart

    You can follow the start-up logs by running the following shell command
    
    .. code-block:: bash

        sudo tail -F -n 300 /var/lib/tomcat8/logs/catalina.out
    

Test GeoServer
===============


Tomcat will extract GeoServer web archive and start GeoServer. This may take some time
Open a web browser (in this example `Firefox`) and navigate to http://example.com:8080/geoserver

GeoNode authentication integration
==================================

All we need to do now is to integrate GeoNode authentication so that GeoNode
administrator will be able to access and administer GeoServer as well.

#. Stop GeoServer

    .. code-block:: bash

        sudo service tomcat8 stop

#. Edit ``/data/geoserver-data/security/filter/geonode-oauth2/config.xml`` with a text editor

    .. code-block:: bash

        sudo vim /data/geoserver-data/security/filter/geonode-oauth2/config.xml

    And make sure the following values are configured as follows:

    .. code-block:: xml

        sudo vim /data/geoserver-data/security/filter/geonode-oauth2/config.xml

        ...
        <accessTokenUri>http://example.com/o/token/</accessTokenUri>
        <userAuthorizationUri>http://example.com/o/authorize/</userAuthorizationUri>
        <redirectUri>http://example.com/geoserver/index.html</redirectUri>
        <checkTokenEndpointUrl>http://example.com/api/o/v4/tokeninfo/</checkTokenEndpointUrl>
        <logoutUri>http://example.com/account/logout/</logoutUri>
         


#. Edit ``/data/geoserver-data/security/role/geonode\ REST\ role\ service/config.xml`` with a text editor

    .. code-block:: bash

        sudo vim /data/geoserver-data/security/role/geonode\ REST\ role\ service/config.xml

    And make sure the following values are configured as follows:

    .. code-block:: xml

        <baseUrl>http://localhost</baseUrl>

#. Edit ``/data/geoserver-data/global.xml`` with a text editor

    .. code-block:: bash

        sudo vim /data/geoserver-data/global.xml

    And make sure the following values are configured as follows:

    .. code-block:: xml

        <proxyBaseUrl>http://example.com/geoserver</proxyBaseUrl>

#. Restart GeoServer to make the changes effective

    .. code-block:: bash

        sudo service tomcat8 restart
